How the web works: front-end, back-end, client & server

Every website lives at the intersection of two worlds. The front-end is everything you see and interact with directly in the browser — layout, text, images, buttons. The back-end is the server-side logic, business rules, and data that power the site behind the scenes.

The client is the device or program that requests information. Your browser — Chrome, Firefox, Edge — is the client that sends requests and renders the HTML it gets back.

The server is a powerful machine or cloud service that stores the website's files (HTML, CSS, JavaScript) and databases, and waits for requests to respond to.

The learning path in front-end starts from this idea: understand how the web works, then learn the languages that build what the client displays — starting with HTML.

كل موقع ويب بُني على فكرتين أساسيتين: الفرونت إند (Front-end) وهو كل شيء تراه وتتعامل معه مباشرة في المتصفح، والباك إند (Back-end) وهو المنطق والبيانات اللي بتشغل الموقع من الخلف.

الـ Client (العميل) هو الجهاز أو البرنامج اللي بيطلب البيانات — المتصفح (Chrome, Firefox, Edge) هو الـ Client اللي بيعرض صفحات HTML.

الـ Server (الخادم) هو جهاز قوي أو خدمة سحابية بتستضيف ملفات الموقع (HTML, CSS, JS) وقواعد البيانات، وبيستقبل الطلبات ويرد عليها.

طريقة الاتصال بينهم بسيطة: المستخدم يفتح المتصفح ويدخل عنوان الموقع → المتصفح يبعت طلب (Request) للسيرفر عبر HTTP/HTTPS → السيرفر يرد بـ استجابة (Response) فيها ملفات HTML وموارد أخرى → المتصفح يعالج البيانات ويعرض الصفحة. مثال: لما تفتح فيسبوك، المتصفح بيطلب الصفحة، والسيرفر بيرجّع ملفات HTML وبياناتك، والمتصفح بيعرضها.

<html> TOPIC 01

What is HTML, and where did it come from?

HTML stands for HyperText Markup Language. It's a markup language, not a programming language — it describes the structure of content on the web: text, links, images, videos, and decoration hooks for CSS.

It was invented by Tim Berners-Lee, often called the "godfather of the web." He also built the first web page and laid the foundation for the World Wide Web itself.

The W3C (World Wide Web Consortium) sets the official standards for HTML. W3Schools is a popular learning site — useful for reference, but not the standards body.

In the 1990s, early browsers like Mosaic popularized the web visually for the first time. Netscape followed and dominated for years before its codebase eventually evolved into what we now know as Firefox (around 2008 onward).

Every HTML element is written as a tag: an opening tag, content, and a closing tag — e.g. <p>text</p>. Some tags are empty / self-closing and never wrap content, like <br> and <img>. HTML tags are not case-sensitive (<HTML> works the same as <html>), but lowercase is the universal convention.

الشرح بالعربي

HTML هي اختصار HyperText Markup Language، وهي لغة توصيفية (Markup Language) وليست لغة برمجة — بتحدد هيكل المحتوى على الويب: نصوص، روابط، صور، فيديوهات، وأماكن للتنسيق بالـ CSS.

اخترعها Tim Berners-Lee، ولُقّب بـ "Godfather to the Web"، وهو نفس الشخص اللي صنع أول صفحة ويب وأساس شبكة الإنترنت العالمية (Web).

W3C هو الجهة المسؤولة عن وضع معايير HTML الرسمية، أما W3Schools فهو موقع تعليمي مشهور للمرجعية فقط، وليس هو الجهة المعيارية.

في التسعينات ظهر متصفح Mosaic وكان أول متصفح اشتهر بعرض الصور والمحتوى بشكل بصري، ثم جاء Netscape وسيطر لسنوات، وبعد ذلك تطور كوده ليصبح متصفح Firefox الحالي (من حوالي 2008).

كل عنصر HTML يُكتب على شكل Tag: تاج فتح، محتوى، وتاج إغلاق — مثل <p>نص</p>. بعض التاجات Empty (مفردة) مالها تاج إغلاق ولا محتوى بينها، زي <br> و <img>. تاجات HTML غير حساسة لحالة الأحرف (<HTML> زي <html>)، لكن الاتفاق العالمي إنها تُكتب بحروف صغيرة.

the basic skeleton of every HTML page

<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My First Page</title>
  </head>
  <body>
    <!-- everything visible goes here -->
  </body>
</html>
<h1> – <h6> TOPIC 02

Headings & page hierarchy

Headings range from <h1> (largest, most important) down to <h6> (smallest). Every page should have exactly one <h1> — usually the page's main title or name.

Search engines treat <h1> as the strongest signal of what the page is about. Getting the heading structure right is one of the most important things a beginner front-end developer can learn — it directly affects SEO (Search Engine Optimization).

A typical profile page is structured like this: <h1> for the person's name, then <h2> for each major section — Profile, Experience, Contacts — with smaller headings or paragraphs nested inside each.

الشرح بالعربي

العناوين تتدرج من <h1> (الأكبر والأهم) إلى <h6> (الأصغر). كل صفحة لازم يكون فيها عنصر <h1> واحد فقط — وعادةً يكون العنوان الرئيسي أو اسم الصفحة.

محركات البحث بتعتبر <h1> أهم إشارة لمعرفة موضوع الصفحة. فهم ترتيب العناوين بشكل صحيح من أهم الأشياء اللي يتعلمها أي مطور Front-end مبدئي، لأنه بيأثر بشكل مباشر على السيو (SEO).

صفحة بروفايل عادة بتُبنى كالتالي: <h1> لاسم الشخص، وبعدها <h2> لكل قسم رئيسي — مثل Profile، Experience، Contacts — وبداخل كل قسم عناوين أصغر أو فقرات.

example: a simple profile structure
<h1>Pascal van Anholt</h1>

<h2>Profile</h2>
<p>Short bio goes here...</p>

<h2>Experience</h2>
<h3>Junior Front-End Developer</h3>

<h2>Contacts</h2>
<p> <br> <hr> TOPIC 03

Paragraphs, line breaks & horizontal rules

<p> wraps a block of text as a paragraph. The browser automatically adds spacing before and after it.

<br> forces a line break inside the same paragraph, without adding the extra spacing a new <p> would. It's an empty tag — no closing tag needed.

<hr> draws a horizontal line across the page, used to visually separate one section of content from another.

الشرح بالعربي

<p> بتلف نص على شكل فقرة، والمتصفح بيضيف مسافة قبلها وبعدها أوتوماتيك.

<br> بتنزل لسطر جديد داخل نفس الفقرة من غير ما تضيف مسافة زيادة كأنها فقرة جديدة. وهي تاج Empty مالها تاج إغلاق.

<hr> بترسم خط أفقي يفصل بين الأقسام بشكل بصري.

p, br and hr together
<p>This is the first line.<br>This is the second line.</p>

<hr>

<p>First section</p>
<hr>
<p>Second section</p>
<a> <img> TOPIC 04

Links & images — and why accessibility matters

<a> creates a hyperlink. The href attribute sets the destination URL, and target="_blank" opens that link in a new tab instead of the current one.

<img> embeds an image. src points to the image file, while alt provides a text description of it.

alt text is not optional decoration: it's read aloud by screen readers for visually impaired users, it's shown if the image fails to load, and it's also an SEO signal.

width and height (in pixels) control the rendered size of the image and help the page avoid layout shifts while it's loading.

Wrapping an <img> inside an <a> turns the whole image into a clickable link.

الشرح بالعربي

<a> بتعمل رابط (Link). الخاصية href بتحدد الوجهة، وtarget="_blank" بتفتح اللينك في تاب جديد بدل ما يفتح في نفس الصفحة.

<img> بتعرض صورة. src هو مكان ملف الصورة، وalt هو وصف نصي للصورة.

النص الموجود في alt ليس مجرد ديكور: قارئات الشاشة (Screen Readers) بتقرأه لذوي الإعاقة البصرية، وبيظهر لو الصورة ما اتحمّلتش، وكمان له تأثير على السيو (SEO).

width و height (بالبكسل) بيحددوا حجم الصورة، وبيساعدوا الصفحة إنها ما تتحرك أثناء التحميل.

لو غلّفت <img> بداخل <a>، الصورة كلها بتتحول لرابط قابل للضغط.

a clickable image, and a plain link
<a href="https://www.linkedin.com/in/yousef-eldbah/">
  <img src="profile.jpg" alt="Yousef Eldbah profile photo" width="200">
</a>

<a href="https://www.google.com" target="_blank">Open Google in a new tab</a>
<ul> <ol> <li> TOPIC 05

Lists — ordered, unordered & nested

<ul> creates an unordered (bulleted) list, and <ol> creates an ordered (numbered) list. Each item — regardless of list type — is wrapped in <li> (list item).

The type attribute on <ol> controls the numbering style. The default is "1" (1, 2, 3...), but it can be changed to letters or Roman numerals:

type="1"1, 2, 3
type="A"A, B, C
type="a"a, b, c
type="I"I, II, III
type="i"i, ii, iii

Lists can also be nested: an <li> can contain its own <ul> or <ol>. This is useful for grouping sub-topics — for example, listing "Margin" and "Padding" as sub-items under "CSS" inside a bigger Front-End list.

الشرح بالعربي

<ul> بتعمل قائمة غير مرتبة (نقاط/Bullets)، و<ol> بتعمل قائمة مرتبة بالأرقام. وكل عنصر في القائمة — في الحالتين — بيُكتب داخل <li>.

الخاصية type في <ol> بتحدد شكل الترقيم. الافتراضي هو "1" (1, 2, 3...)، ويمكن تغييره لحروف أو أرقام رومانية كما في الجدول بالأعلى.

القوائم ممكن تكون متداخلة (Nested): عنصر <li> يقدر يحتوي على <ul> أو <ol> جديدة بداخله — مفيد لتجميع مواضيع فرعية، مثل عمل "Margin" و"Padding" كعناصر فرعية تحت "CSS" داخل قائمة Front-End الأكبر.

a nested ordered/unordered list
<ol type="I">
  <li>HTML</li>
  <li>CSS
    <ul>
      <li>Margin</li>
      <li>Padding</li>
    </ul>
  </li>
  <li>JavaScript</li>
</ol>
<table> TOPIC 06

Tables — structuring tabular data

<table> holds tabular data. <thead> groups the header row(s), and <tbody> groups the body rows.

Each row is a <tr> (table row). Header cells use <th> (table header), and regular data cells use <td> (table data).

The legacy border attribute adds a visible grid around the table — in modern HTML this is usually handled with CSS instead, but it's still useful for quick prototypes.

الشرح بالعربي

<table> بتعمل جدول لعرض بيانات. <thead> فيه عنوان (أو عناوين) الجدول، و<tbody> فيه البيانات نفسها.

كل صف بيُكتب في <tr> (Table Row)، وخلايا العنوان بتُكتب في <th> (Table Header)، أما خلايا البيانات العادية بتُكتب في <td> (Table Data).

الخاصية القديمة border بتضيف خطوط حول الجدول — في HTML الحديثة الأفضل التحكم فيها بـ CSS، لكنها مفيدة في النماذج السريعة.

a simple table with header and body
<table border="2">
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>Job</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Ahmed</td>
      <td>25</td>
      <td>Developer</td>
    </tr>
  </tbody>
</table>
<dl> <dt> <dd> TOPIC 07

Definition lists

<dl> wraps a definition list — a set of terms and their descriptions.

<dt> (definition term) holds the term, and <dd> (definition description) holds its explanation, placed right below it.

This is handy for glossaries, FAQs, or any term/description pairs.

الشرح بالعربي

<dl> بتعمل قائمة تعريفية — مجموعة من المصطلحات وتعريفاتها.

<dt> هو المصطلح نفسه، و<dd> هو شرح المصطلح اللي بييجي تحته مباشرة.

مفيدة جدًا في قوائم المصطلحات أو الأسئلة الشائعة.

a small glossary
<dl>
  <dt>HTML</dt>
  <dd>Language used to structure web pages</dd>

  <dt>CSS</dt>
  <dd>Language used to style how pages look</dd>
</dl>
<head> TOPIC 08

The <head> section & SEO basics

The <head> holds information about the page that isn't displayed directly in the body: the <title>, links to CSS/JS files, and <meta> tags.

  • <title> — sets the text shown on the browser tab. Keep it roughly under 60 characters so it doesn't get cut off in search results.
  • <meta name="description" content="..."> — the snippet often shown under your page's title in search results. Aim for roughly 150–160 characters.
  • <meta name="keywords" content="..."> — relevant keywords for the page. Less influential today, but still used.
  • <meta name="author" content="..."> — credits the page's creator.

الشرح بالعربي

<head> فيه معلومات عن الصفحة، ومش معروضة مباشرة في الـ body: زي <title> وربط ملفات CSS/JS وتاجات <meta>.

  • <title> — النص اللي يظهر في تبويب المتصفح (Tab). يُفضّل ألا يزيد عن 65 حرف تقريبًا.
  • <meta name="description"> — الوصف اللي يظهر تحت عنوان الصفحة في نتائج البحث. يُفضّل حوالي 150–160 حرف.
  • <meta name="keywords"> — كلمات مفتاحية تخص الصفحة، تأثيرها أقل الآن لكنها لسه تُستخدم.
  • <meta name="author"> — اسم صانع الصفحة.

أهم عوامل السيو (SEO) اللي أخذناها لحد دلوقتي: 1) ترتيب العناوين، 2) نص alt للصور، 3) عنوان الصفحة Title، 4) وصف الميتا Meta Description.

a well-described <head>

  <title>Yousef Eldbah — Full Stack Developer</title>
  <meta name="description" content="Portfolio and notes from the Mindloop Academy front-end track.">
  <meta name="keywords" content="HTML, CSS, JavaScript, Front-End">
  <meta name="author" content="Yousef Eldbah">
</head>
<br/> TOPIC 09

Empty tags vs nested tags — quick recap

Empty (self-closing) tags have no closing tag and no content between them — like <br>, <img>, and <hr>.

Nested tags are elements placed inside other elements — like a <ul> inside an <li>, or an <img> inside an <a>. Nesting is how the page builds its overall structure tree.

الشرح بالعربي

التاجات الـ Empty ملهاش تاج إغلاق ولا محتوى بينها — زي <br> و <img> و <hr>.

التاجات الـ Nested (المتداخلة) هي عناصر موجودة جوه عناصر أخرى — زي <ul> جوه <li>، أو <img> جوه <a>. التداخل ده هو اللي بيبني شجرة (Tree) هيكل الصفحة كلها.

Build your personal profile page

Using only the tags from this session, build a single HTML page that works as your personal profile / portfolio intro. Save it as one .html file and send it over for review.

Checklist

  • One <h1> with your name, and <h2> sections for About, Skills, and Contact.
  • At least one <p> describing yourself — use <br> where a manual line break makes sense.
  • A profile photo with <img> (meaningful alt text), optionally wrapped in an <a> linking to one of your social profiles.
  • A skills list using <ul>, with at least one <ol> somewhere (try a non-default type).
  • A <table> listing tools/technologies and your experience level for each.
  • A small <dl> glossary with 2–3 terms relevant to your field.
  • A complete <head> with a descriptive <title>, plus description, keywords and author meta tags.

قائمة التسليم

  • عنصر <h1> واحد باسمك، وعناصر <h2> لأقسام About، Skills، Contact.
  • فقرة <p> على الأقل تتكلم عن نفسك — واستخدم <br> لو احتجت سطر جديد داخل نفس الفقرة.
  • صورة بروفايل بـ <img> (مع alt واضح)، ويمكن تغليفها بـ <a> تروح لأحد حساباتك.
  • قائمة مهارات بـ <ul>، مع <ol> واحدة على الأقل بنوع ترقيم غير الافتراضي.
  • جدول <table> فيه الأدوات/التقنيات ومستوى خبرتك في كل واحدة.
  • قائمة تعريفية <dl> فيها 2-3 مصطلحات تخص مجالك.
  • <head> كامل فيه <title> واضح، مع meta tags لـ description وkeywords وauthor.
